Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

Question 3 - Training and Testing¶

  • What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?

Hint: Think about how overfitting or underfitting is contingent upon how splits on data is done.

Answer:

Splitting a dataset into a testing and training subset is necessary to check for over- and underfitting. Overfitting occurs if a model is closely adapted to the training data and does not generalize to unkown data (or testing data). Underfitting occurs if a model is too simple and hence doesn't caputure the patterns in the data. By splitting a dataset into a testing and training subset a machine learning engineer can test for over- and underfitting. If a model shows good performance on the training data and bad performance on the test data the model overfits. If a model shows bad performance on the training data and bad performance on the test data the model underfits. The size of the testing dataset is usually smaller than the training set. It is also a golden rule never to use the testing set for training.

Question 7 - Grid Search¶

  • What is the grid search technique?
  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer:

  • What is grid search?: Supervised machine learning algorithms very often have multiple parameters that can be adjusted by a machine learning engineer. For example, support vector machines have the parameters gamma and C which change the behavior of the underlying algorithm. A machine learning engineer might train multiple such models and change these parameters from model to model. However, this approach would resemble a trial-and-error approach. A better way to formalize this problem is to use the grid search technique. With grid search you define the parameters of an algorithm before hand and then run the algorithm multiple times with the defined parameters. For example, if you would train a support vector machine and want to tune the parameters gamma and C you could first define two values of gamma and two values of C (see here). What you would get is a grid of two rows and two columns. The rows might indicate the values of gamma and the columns might indicate the values of C. With a two and two grid the grid search technique would have to run the support vector machine four times.
  • Optimize a learning algorithm: From running an algorithm multiple times with the a priori defined parameter values you are able to select the algorithm which provides the best evaluation metrics. In our example we might find that the support vector machine with a gamma of 2 and a C of 100 yields the highest F1-score. Thus, we could be fairly confident that this algorithm performs best on new data.

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer:

  • What is k-fold cross-validation?: When evaluating an algorithm with multiple paramaters on a test set, a machine learning engineer might be tempted to compare the evaluation metrics on the test set and then decide upon the best model. I could, for example, tweak a parameter on an algorithm 4 times and then calculate my evaluation metric for each of the models. By doing so, knowledge about the test set "leaks" into the model and its evaluation. Such a "leak" would break the golden rule of splitting a dataset in a test and training set: Never use the testing set for training. A solution to this problem might be the use of a validation set. Instead of two datasets you would have three: a training set, a testing set and a validation set. In my example, I would have to train my model, test it on the validation set and then report the evaluation metric of my model from the testing set. However, this procedure would result in a reduced training set since I would have to split the dataset one more time. A solution to this problem is the k-fold cross-validation technqiue. In k-fold cross-validation the training set is split into k-bins of equal size. For example, I could split a training set of 100 datapoints into five bins of 20 datapoints. Each bin is used as a testing set once. So, if I have a training set of five bins I would generate five testing sets. The remaining bins are used for training my model. Hence, you can run your algorithm k times and then average the resulting evaluation metric of each iteration. The benefit of using k-fold cross-validation is that you are able to validate your model on an evaluation metric without touching the testing set. However, k-fold cross-validation is computationally more expensive and takes more time.
  • Benefit for grid search: The idea behind the grid search technique is to find the model that provides the best evaluation metric on the testing set. Again, doing so would break the golden rule of splitting a dataset into a test and training set. Information about the testing set could leak into the evaluation of our model. K-fold cross-validation can solve this problem because we can calculate the evaluation metric of a model and compare them without looking at the testing set. By averaging the evaluation metric through k-fold cross-validation we can be fairly confident, which of the tuned models is the best and calculate it's predicitve power using our evaluation metrics on the testing set.

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    # cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': range(1, 10)}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor, params, scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
